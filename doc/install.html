Trull: Installation and Usage Instructions

System Requirements

Trull requires Java 2 Standard Edition (J2SE), preferably J2SE 1.3 or newer.

Trull also requires Apache Ant 1.6.0 or newer for building from source or running the demos locally (optional).

Trull also requires the following libraries and frameworks, which are already included in binary form in this distribution:

  • util.concurrent
  • JUnit
  • Log4j

Optionally, Trull uses the following libraries and frameworks, which are also included in binary form in this distribution:

  • Spring Framework (spring-core.jar and spring-beans.jar only)
  • Jakarta Commons logging (required by the Spring Framework)

The Trull team is grateful to the developers of these high-quality frameworks.

Installation

Unzip trull-VERSION.zip or trull-VERSION.tar.gz in a suitable directory. None of the directory names involved should contain spaces.

Usage

If you do not have Apache Ant, you can run the demos using Java Web Start through the API documentation.

If you do have Ant, you can run the demos from the top-level project directory like so:

    ant demo-battleship
    ant demo-counter1
    ant demo-counter1a
    ant demo-counter2
    ant demo-counter3
    ant demo-counter4
    ant demo-counter5
    ant demo-counter6
    ant demo-microwave
    ant demo-office
    ant demo-rmi1        # run each of demo-rmi1 and demo-rmi2 in a
    ant demo-rmi2        # different terminal on the same machine
    ant demo-stopwatch
    ant demo-wristwatch
    ant demo-tasktimer
    ant demo-tasksimple
    ant demo-taskcomplex
    ant demo-swingtester

For complete documentation of the Trull Java API and demos, please refer to the API documentation.

Trull will soon include an XML-based front end.

Building from Source (Optional)

  • To build the project: 
        ant
  • To regenerate the javadoc: 
        ant javadoc
  • To create a full distribution: 
        ant dist
  • To clean the build tree: 
        ant clean

Using Trull in your own Projects

You should include the following libraries in the project:

  • lib/trull.jar
  • lib/concurrent.jar
  • lib/log4j.jar

Using Trull with the Spring Framework

In addition, you should include the following libraries:

  • lib/spring-core.jar
  • lib/spring-beans.jar
  • lib/commons-logging.jar

Please see the wristwatch demo for details.
